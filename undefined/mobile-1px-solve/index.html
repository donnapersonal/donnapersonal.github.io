移动端 1px 的解决方案

发表于2021-05-15|更新于2025-02-09|前端笔记
|浏览量:

背景

在高清屏（Retina）下移动端的 1px 会很粗，如下面图一是假的 1 像素，图二是真的 1 像素

产生原因

主要是跟 DPR(device Pixel Ratio) 设备像素比有关，它是默认缩放为 100% 的情况下设备像素和 CSS 像素的比值 window.devicePixelRatio = 物理像素 / CSS 像素

主流的屏幕 DPR=2 (iPhone 8)、3 (iPhone 8 Plus)，拿 2 倍屏来说，设备的物理像素要实现 1 像素，而 DPR=2，所以 CSS 像素只能是 0.5

一般设计稿是按照 750 来设计的，它上面的 1px 是以 750 来参照的，而写 CSS 样式是以设备 375 为参照的，所以应该写 0.5px 即可，iOS 8+ 系统支持，安卓系统不支持

解决方案

WWDC 对 iOS 系统给出的方案

直接设置 0.5px 的边框

问题：Retina 屏的浏览器可能不认识 0.5px 的边框，而是会把它解释成 0px 没有边框，包括 iOS 7 和之前版本、OS X Mavericks 及之前版本，Android 设备在 3 倍屏下不是 0.3333px；在 Chrome 上模拟 iPhone 8Plus，发现小于 0.46px 时是显示不出来

总结：

  • 优点：简单，没有副作用
  • 缺点：支持 iOS 8+，不支持安卓，后期安卓支持就好

解决方案：通过 JS 检测浏览器能否处理 0.5px 的边框，若可以则给 <html> 元素添加个 class。看起来是个解决方案，可要兼容安卓设备，和 iOS 8 以下设备怎么办？这个思路行不通

使用边框图片

图片的颜色就是此后 border 的颜色

总结

  • 优点：没有副作用
  • 缺点：想要修改 border 颜色，需重新修改图片；圆角会比较模糊

使用 box-shadow 实现

前面两个值 x，y 主要控制显示哪条边，后面两值控制的是阴影半径、扩展半径

总结：

  • 优点：使用简单，圆角也可实现
  • 缺点：模拟的实现方法，仔细看可看出来这是阴影不是边框

淘宝 M 站是通过 viewport + rem 实现的

在 devicePixelRatio = 2 时，输出 viewport

在 devicePixelRatio = 3 时，输出 viewport

同时通过设置对应 viewport 的 rem 基准值，这种方式就可像以前一样轻松愉快的写 1px

总结：

  • 优点：全机型兼容，直接写 1px 更方便
  • 缺点：适用于新的项目，老项目可能改动大

伪类 + transform

原理：把原先元素的 border 去掉，然后利用 :before 或 :after 重做 border，并将 transform 的 scale 缩小一半，原先的元素相对定位，新做的 border 绝对定位

单条 border

四条 border

总结：

  • 优点：全机型兼容，实现了真正的 1px，且可以圆角
  • 缺点：暂用了 after 伪元素，可能影响清除浮动；<td> 用不了
  • 可参照这个 border 生成器 写一个 scss 的 1px 边框生成器，该生成器考虑了三种情况：单边边框、多边边框、边框的圆角
